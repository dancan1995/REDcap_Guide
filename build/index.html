REDCap Guide

Introduction

Welcome to REDCap 101 Training! This guide will walk you through building projects in REDCap.

Objectives:

  • Learn where to access training materials.
  • Understand the basics of project setup and user rights management.
  • Learn to optimize your project using advanced features.

Step 1: Setting Up Your Project

1. Click the +New Project tab and complete the form with project details.

2. Define whether your project will be a Classic Database or Longitudinal Study. Consider how your data will be collected and organized over time.

3. Configure survey settings and enable longitudinal options as applicable.

4. Ensure youâ€™ve specified the principal investigator (PI) and study team members.

Step 2: Building Data Collection Forms

1. Use the Online Designer to add or modify data collection fields in your project. This includes text fields, dropdowns, and date inputs.

2. Leverage features like field validation, branching logic, and action tags to ensure accurate data entry.

3. Follow naming conventions for fields to maintain clarity and consistency. For example, use short but meaningful names like birth_date instead of overly descriptive or generic labels.

4. Test your forms by entering sample data to ensure they function as expected.

Step 3: Managing User Permissions

1. Assign user rights either individually or based on predefined roles for secure access. Roles can include data entry, data analysis, and project design.

2. Use Data Access Groups (DAGs) to manage multi-center collaborations, ensuring users only access relevant data.

3. Regularly review and update permissions to maintain compliance with data security policies.

Step 4: Testing and Moving to Production

1. Test your project thoroughly by entering sample data and reviewing results. Check for errors in field validations, branching logic, and calculations.

2. Export test data to validate formatting and usability for analysis. Ensure any sensitive information is correctly de-identified if required.

3. After testing, submit your project for production status. This step locks the design and enables live data collection.

Step 5: Leveraging Advanced Features

1. Use action tags to automate field behaviors, such as auto-filling fields based on prior responses or disabling fields under certain conditions.

2. Apply Smart Variables to dynamically reference project information, such as event names, user IDs, or timestamps, across different forms or events.

3. Configure longitudinal events and designate instruments for flexible data collection over multiple time points.

4. Explore advanced survey features like automated invitations, survey queues, and conditional survey logic.

Step 6: Utilizing Training Resources

1. Access training videos and tutorials available within REDCap under the "Help & FAQ" section.

2. Review the REDCap documentation for detailed guidance on advanced features and troubleshooting tips.

3. Participate in live training sessions or webinars hosted by your institution or REDCap support team.

Step 7: Maintaining and Updating Your Project

1. Regularly monitor your project for data quality issues, such as missing or inconsistent entries.

2. Update your project as needed, but be cautious when making changes to forms or logic that could impact existing data.

3. Archive completed projects or export data for long-term storage and analysis.
