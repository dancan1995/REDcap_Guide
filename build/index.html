REDCap Training Guide

1. Home Page

REDCap (Research Electronic Data Capture) is a secure web platform designed for building and managing online surveys and databases. It streamlines data collection for research and quality improvement projects, offering tools for secure data entry, project management, and statistical analysis.

At KCON, REDCap is widely used for research and clinical studies, enabling users to design surveys, track data securely, and generate reports efficiently. With features like automated export to Excel, SPSS, and R, as well as mobile data collection options, REDCap supports various data-driven initiatives.

This tutorial will guide you through setting up and managing your REDCap projects, ensuring compliance with research protocols and institutional guidelines.

For access and support, visit GVSU REDCap Support or contact IT Services.

2. Creating a New Project

  • Click "+ New Project" in the top menu to start a new project.
  • Enter a Project Title in the designated text field.
  • Select a Project Purpose from the dropdown menu, with options such as:
    • Practice / Just for fun
    • Operational Support
    • Research
    • Quality Improvement
    • Other
  • Choose a Project Creation Option, such as starting with a blank template or uploading an existing XML file.
  • Select a Project Template if applicable, with options like Basic Demography, Classic Database, Longitudinal Database, and more.
  • Click "Create Project" to finalize the setup.

3. Designing the Project

  • Main project settings: Options to enable surveys, longitudinal data collection, and MyCap. Users can modify the project title and purpose.
  • Design your data collection instruments: Tools like Online Designer and Data Dictionary help create and manage data collection forms.
  • Enable optional modules and customizations: Features like repeating instruments, auto-numbering for records, and communication tools (email and SMS alerts).
  • Set up project bookmarks: Allows users to create shortcuts for quick access to external or internal project-related pages.
  • User Rights and Permissions: Controls access levels, letting admins grant permissions to different users based on roles.

4. Creating a Data Collection Form

To create a data collection form in REDCap, follow these steps:

  • Go to your REDCap project and navigate to the Online Designer tab.
  • Under Data Collection Instruments, click Create to add a new form from scratch.
  • Alternatively, use the Import option to upload an instrument ZIP file from another project.
  • Locate the newly created form under the Instrument Name column.
  • Ensure the form has at least one field, as shown in the Fields column.
  • Click the Enable button under the Enabled as survey column if you want to use it as a survey.
  • Use the available options to configure form settings, such as PDF Snapshots and Survey Queue.
  • Once configured, start adding fields and questions to the form.

5. Continuing with the Online Designer

After creating a data collection form, follow these steps to continue editing it:

  • Navigate to the Online Designer tab.
  • Watch the Video Guide for additional help.
  • Ensure you are working on the correct form under Current Instrument.
  • Click Add Field to start adding data fields to the form.
  • Edit or delete existing fields by clicking on them.
  • Use the drag-and-drop feature to reorder fields easily.
  • Changes are saved in real-time while the form is in development.

6. Adding New Fields in the Form

To create a new field, click on the "Add New Field" button. You can choose from various field types based on the kind of data you need to collect.

  • Field Types Video: Watch this for an overview of available field types.
  • Field Type Selection: Options include text, multiple-choice, date, file upload, and more.
  • Identifier Checkbox: Mark this if the field collects sensitive or personal information.

7. Field Creation & Results

7.1 Field 1

After selecting a field type, configure its properties to ensure accurate data collection.

  • Field Type Dropdown: Select the appropriate data format (e.g., Text, Date, Number).
  • Field Label: Provide a clear name (e.g., "Date of Birth").
  • Variable Name: This is the system’s reference name (e.g., "what_is_your_date_of_birth").
  • Validation: Choose a format to ensure proper data entry (e.g., "M/D/Y" for dates).

Once the field is added, review and manage it as needed.

  • Field Actions: Edit, delete, or reposition the field within the form.
  • New Field Display: View how the field appears in the form after saving.
  • Add More Fields: Click to add additional fields for more data collection.

7.2 Field 2

Adding a Drop-Down List Field

When adding a new field, select the following:

  • Field Type: Choose Multiple-Choice Drop-Down List (Single Answer).
  • Field Label: Enter "What is your place of birth?" to describe the question.
  • Choices: Define the available options:
    • USA
    • UK
    • Canada
    • Germany
    • China

Configuring Field Properties

Ensure the following settings are applied:

  • Variable Name: Use "place_of_birth" as the system’s reference.
  • Required: Select Yes or No based on whether the field is mandatory.
  • Identifier: Choose Yes if it contains personal information.
  • Custom Alignment: Set to Right / Vertical (EV) for proper display.
  • Enable Auto-Complete: This helps users type and select choices quickly.

After setting up the field:

  • Click "Save" to finalize the field.
  • View the field in the form preview.
  • The drop-down list will show the configured country choices.

7.3 Field 3

Adding Health Insurance

  • Field Type: Yes-No
  • Field Label: "Do you have health insurance?"
  • Choices: 1 = Yes, 0 = No
  • Variable Name: health_insurance
  • Required: Yes/No (Based on project needs)
  • Custom Alignment: Right / Vertical (EV)
  • Click Save ✅

BP & Sugar Level Matrix

  • Matrix Header: "BP and Sugar level Measurements"
  • Matrix Rows:
    • Blood pressure measured (bp_measure)
    • Sugar level measured (sugar_level)
  • Answer Format: Yes-No Radio Buttons
  • Matrix Group Name: metrics
  • Click Save ✅

Variable Name: record_id

Description: A unique identifier for each participant or entry in the database.

Variable Name: dob

Description: Captures the participant's birth date, which may be used for age calculations.

Variable Name: place_of_birth

Description: A multiple-choice field where participants select their place of birth (e.g., USA, UK, Canada, etc.).

Variable Name: health_insurance

Description: A Yes/No field that asks participants if they have health insurance.

Matrix Header: "BP and Sugar level Measurements"

Matrix Variables:

  • Blood Pressure Measured (bp_measure): A Yes/No response indicating whether blood pressure was measured.
  • Sugar Level Measured (sugar_level): A Yes/No response indicating whether blood sugar level was measured.

Answer Format: Yes-No radio buttons for each row.

Matrix Group Name: metrics

The individual fields (Date of Birth, Place of Birth, Health Insurance) are designed to collect essential participant details.

The matrix fields allow efficient grouping of health-related measurements in a structured format, ensuring easy data entry and analysis.

This setup ensures that the data is collected in a systematic and user-friendly way, optimizing data consistency and accuracy for analysis.

The matrix structure efficiently organizes participant responses, making it easy to analyze satisfaction levels regarding health condition monitoring.

The format ensures that responses are standardized, allowing for streamlined data collection and assessment.

8. Using the Data Dictionary

Description: The Data Dictionary in REDCap allows users to modify or create fields using a structured CSV file. This feature is useful for bulk editing or adding project variables efficiently.

Steps to Use the Data Dictionary:

  1. Navigate to the Data Dictionary tab.
  2. Download the current Data Dictionary in CSV format.
  3. Edit the CSV file to modify field properties.
  4. Upload the updated Data Dictionary back into REDCap.

Upload Requirements:

  • File format: CSV
  • Ensure that field names are correctly formatted.
  • Date fields must follow YYYY-MM-DD or YYYY-MM-DD HH:MM format.

9. Codebook Overview

Description: The Codebook provides a human-readable summary of the project's Data Dictionary, displaying field names, labels, types, and validation rules.

Key Features of the Codebook:

  • Lists all fields and variables in a structured format.
  • Includes field attributes such as text validation, dropdown choices, and required fields.
  • Allows searching for specific variables using keywords.

The Codebook and Data Dictionary streamline project management by offering an organized way to view, edit, and structure research variables within REDCap.
